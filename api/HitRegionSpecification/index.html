interface HitRegionSpecification

Specifies which properties a hit region responds too.

Properties
click, contextMenu, cursor, doubleClick, grab, grabEnd, id, keyDown, keyUp, mouseDown, mouseEnter, mouseLeave, mouseMove, mouseUp, parentId, wheel

Properties

click
optional

Callback when this region is clicked.

Type:
(event: MouseHitEvent) => void
contextMenu
optional

Callback when the user attempts to open a context menu. (usually by right click).

Type:
(event: MouseHitEvent) => void
cursor
optional

Cursor on hover.

Type:
string
doubleClick
optional

Callback when this region is double-clicked.

Type:
(event: MouseHitEvent) => void
grab
optional

Callback while a grab of this region is going on.

Grab events trigger when a click-and-drag is recorded on the hit region.

Type:
(event: GrabHitEvent) => void
grabEnd
optional

Callback when a grab has ended that was initiated on this region.

Type:
() => void
id

Unique identifier for this region.

While it is preferred to make hit regions outside of the draw loop, it is allowed and the ID is what establishes conceptually a unique region.

Type:
string

optional

Callback when a key is pressed.

Type:
(event: MouseHitEvent) => void
mouseLeave
optional

Type:
(event: MouseHitEvent) => void
mouseMove
optional

Callback when the mouse moves over this region (or any child regions).

Type:
(event: MouseHitEvent) => void
mouseUp
optional

Callback when a mouse-up occurs on this region.

Type:
(event: MouseHitEvent) => void
parentId
optional

Optional identifier of a parent region. Some region properties ‘bubble up’ to parent regions.

For example if a cursor is set on a parent, but not on the child region, the parent’s cursor will be displayed even if the child is hovered.

Type:
string
wheel
optional

Callback when a wheel event occus on this region.

(event: WheelHitEvent) => void
